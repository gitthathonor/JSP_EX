*Ajax(Asynchronous JavaScript and XML) : 비동기 방식의 자바스크립트와 XML

 동기 방식 (페이지 이동이 필요하다/화면전환이 있음)
 	사용자 -> 웹서버(was) -> 사용자
 	
 비동기 방식 (페이지 이동 없이 사용/화면전환이 없음)
 	사용자 <-> 웹서버(was)
 	
 	*ajax 사용하기
 	
 	$.ajax({
 		url:"전송페이지(액션페이지)",
 		type:"전송방식(get,post)",
 		data:"전송할 데이터"
 		dataType:"요청한 데이터타입(html,xml,json,text)",
 		success:function(성공시 가져오는 데이터){
 			전송 성공시 처리할 문장들
 		}
 		
 	});
 	
 	
 	* Ajax의 요청 응답 타입
 	
 	HTML
	 	
 	XML - html과 유사함.(태그를 사용해서 데이터를 표현)
 		장점 : 복잡한 구조의 데이터 표현 가능,
 				다른 프로그램과의 데이터 교환 가능(사용빈도 높음),
 				Dom-html 문법 사용가능ㅇ
 		단점 : 데이터값 제외한 나머지 데이터가 많음.(전송량이 많다 -> 처리속도가 느림),
 				요청이 반드시 동일 페이지(같은 도메인에서 실행)
 		
 	JSON - 자바스크립트 객체 표현식(JavaScript Object Notation)
 			=> 자바스크립트 문법
 			
 			장점 : 다른 도메인에서 페이지 요청 처리 가능
 				html/xml 코드보다 훨씬 간결,
 				js와 같이 사용(웹 애플리케이션에서 사용)
 			
 			단점 : 문법에 예민함.(따옴표, 콤마, 콜론, 세미콜론 사용주의)
 				js기반의 데이터이기 때문에 악의적인 데이터 생성이 얼마든지 가능하다.
 				
 				
 			웹 - html / xml /json
 			애플리케이션 - xml/json/CSV
 				=> 개방형 표준 데이터 포맷
 				
 			* 데이터를 key/value 쌍으로 표시하는 형태 (단순 텍스트 - 직관적)
 			* 상대적으로 빠른 데이터 전송
 			* 특정 언어, 플랫폼에 독립적이므로, 형식만 지켜서 사용할 경우 어디든 사용가능
 			
 			
 			* 문법
 			0) "속성(key)" : 값(value) 쌍 (콤마로 데이터 구분)
 			1) { } - JSON 객체
 			2) [ ] - JSON 배열
 			
 			3) 정수(10진수), 실수(고정 / 부동소수점), Boolean, null,
 				문자열("0개이상의  유니코드"), 배열
 				
 				ex)
 				나이 20살
 				
 				JSON
 				1) "age":20 //JSON
 				2) {"age":20,"name":kim}
 				
 				
 				XML
 				1)<age>20</age> 
 				2)<person>
 					<age>20</age>
					<name>kim</name> 				
 				</person>

 	Java코드를 JSON타입으로 변경하는 라이브러리
 	(JSON Object, JSON Array로 사용)

페이지 정보전달방식

=> 폼태그에 있는 데이터(input태그)를 전달하기 위한 형태

GET방식 /get()

1) 전송할 데이터(파라미터)를 URL정보에 붙여서 전송o
=>주소?파라미터명=값&파라미터=값 (외부에서 확인 o)
2) 길이의 제한이 있음(256byte)
3) 주소줄에 데이터를 담아서 전달 4) 데이터 처리 속도가 빠름 why? 길이의 제한과 관련, 어차피 post와 속도 차이는 엄청 나진 않는다. 5) 모든데이터가 주소에 보임 => 보안적인 문제발생 6) 대표적인 겟방식은 일반적인 정보창

POST방식/ post()

1) 전송할 데이터(파라미터)를 URL정보에 붙여서 전송x
=> BODY에 담아서 전달(외부에서 확인 x)
2) 길이의 제한이 없음 3) 데이터를 body에 담아서 전달, body에 대해서는 추후 jsp를 다룰 때 다시 설명 4) 데이터 처리 속도가 느림 5) 모든 데이터가 주소에 x => 보안적으로 우수함 6) 대표적인 포스트 방식은 로그인
